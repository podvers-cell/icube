## ٢. ما يجب إنجازه لاحقاً 📋

### من تقرير التدقيق – تحسين لاحق (Improve later)
| البند | الأولوية | الوصف |
|--------|----------|--------|
| قواعد أمان Firestore | عالية | التأكد من أن القراءة/الكتابة لمجموعات الداشبورد مقيدة بصلاحيات الأدمن فقط (يُنفَّذ يدوياً في Firebase Console). |
| تحديد معدل الطلبات (Rate limiting) | متوسطة | منع إساءة استخدام النماذج وواجهة الرفع (مثلاً عبر Vercel middleware أو serverless). |
| روابط canonical و Open Graph | متوسطة | إضافة لجميع الصفحات العامة لتحسين SEO ومشاركة الروابط. |
| اختبارات E2E | متوسطة | تغطية مسارات: الحجز، التواصل، تسجيل الدخول → الداشبورد (مثلاً Playwright أو Cypress). |
| تقسيم المكونات الكبيرة | منخفضة | تقسيم Studio.tsx و VideoPlayerModal.tsx لتحسين الصيانة. |
| تحميل الداشبورد كحزمة منفصلة | منخفضة | Lazy load أو code splitting للداشبورد لتحسين أداء الصفحة الرئيسية. |
| طبقة نماذج/تحقق مشتركة | منخفضة | React Hook Form أو تحقق موحّد على الواجهة باستخدام نفس مخططات Zod. |
